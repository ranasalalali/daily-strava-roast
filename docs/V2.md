# Daily Strava Roast V2

## Goal

Move from template-first roast generation to a hybrid architecture that keeps deterministic data handling but uses a language model for the final roast paragraph.

## Why V2 exists

V1 is reliable, portable, and testable, but its prose still has a visible template ceiling.

V2 should aim for:
- more natural writing
- stronger variation
- better use of context
- less sentence-fragment assembly

## Core idea

Split the system into two layers.

### Layer 1: deterministic roast context builder
This layer should:
- fetch Strava activities
- roll up activity by day
- compute useful facts and patterns
- track recent roast memory
- decide tone/spice and output constraints
- produce a compact structured roast context

### Layer 2: model-backed paragraph writer
This layer should:
- read the roast context
- produce one short paragraph
- keep tone sharp, funny, and readable
- avoid bullet points and stat-dumping
- avoid repeating recent joke families when possible

## Proposed flow

1. Fetch + summarize daily activity
2. Build roast context JSON
3. Add recent roast memory summary
4. Build a constrained prompt
5. Generate one roast paragraph with the model
6. Optionally fall back to V1 deterministic roast if model generation is unavailable

## Suggested constraints for the model

- 1 short paragraph
- funny, dry, slightly mean but not cruel
- do not invent stats
- do not output bullets
- avoid repeating exact phrasing from recent roasts
- prefer natural language over dashboard-like reporting

## Fallback strategy

If model generation fails or is unavailable:
- fall back to the V1 deterministic roast path

This keeps scheduling and daily usage robust.

## Open questions

- which model/runtime should generate the roast?
- should the model run locally or via the active agent runtime?
- how much roast memory should be exposed in the prompt?
- should repeated manual invocations increase variation more than scheduled daily roasts?

## Release target

Develop on the `v2` branch and target a future `0.2.0` release once the model-backed roast is clearly better than V1.

## Current progress

- Added a first deterministic `context` mode that emits a structured roast context JSON for the latest day.
- Added a focused context-builder module and test to keep V2 grounded in clean intermediate data before any model-backed paragraph generation.
- Added a constrained `prompt` builder and a lightweight `preview` writer so the V2 path can be evaluated without immediately replacing the stable roast path.
- Added a generation boundary for model-backed writing.
- Clarified that connected/default-model generation belongs to the OpenClaw runtime layer, while the package/CLI should remain deterministic-first and provide stable V1 fallback behavior.
